Routing is the ability to navigate within different parts of an application when a user enters a URL or clicks an element such as link, button or image within the application.<\/p>\n\n\n\n

What is react router?<\/h2>\n\n\n\n

React router is a routing library built on top of the react which is used to create the routing in react apps. React Router conditionally render certain components to display depending on the route<\/em> being used in the URL (\/<\/code> for the home page, \/about<\/code> for the about page, etc.).<\/p>\n\n\n\n

In the below example, we are using the BrowserRouter<\/strong><\/code> with the assumption that the server running the application is dynamic.<\/p>\n\n\n\n

Open the App.js file and import the components. In this example we have Home, About and Contact.<\/p>\n\n\n\n
React router gives us four components Route, Link, BrowserRouter and Switch which help us to implement the routing. You\u2019ll need to import BrowserRouter<\/code>, Route<\/code>, and Switch<\/code> from react-router-dom<\/code> package:<\/p>\n\n\n\n

The <Route\/><\/strong><\/code> component is one of the most important building blocks in the React Router package. It renders the appropriate user interface when the current location matches the route\u2019s path<\/strong><\/code>. In the Route component, we need to pass the two props, one is path and second component. The path<\/strong><\/code> describes the pathname that the route should match. The component prop describes the component user needs to see when they will navigate to that path.<\/p>\n\n\n\n
The react-router library also contains a <Switch\/><\/strong><\/code> component that is used to wrap multiple  <Route\/><\/strong><\/code>  components. The Switch<\/strong><\/code> component only<\/strong> picks the first<\/strong> matching route among all its children routes.<\/p>\n\n\n\n
Now if you enter manually localhost:3000\/about you will see About component is rendered.<\/p>\n\n\n\n
Adding Navigation using Link component<\/h2>\n\n\n\n
The react-router package also contains a <Link\/><\/strong><\/code> component that is used to navigate the different parts of an application by way of hyperlinks. It is similar to HTML\u2019s anchor element but the main difference is that using the Link<\/strong><\/code> component does not reload the page but rather, changes the UI. Using an anchor tag would require that the page is reloaded in order to load the new UI. When the Link<\/strong><\/code> component is clicked, it also updates the URL.<\/p>\n\n\n\n
