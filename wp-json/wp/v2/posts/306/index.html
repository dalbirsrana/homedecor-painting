Progressive Web App (PWA) <\/strong>is application software delivered through the web, uses web browser APIs and features along with progressive enhancement approach to bring a native app-like user experience to cross-platform web applications. It is intended to work on any platform that uses a standards-compliant browser. PWA can be thought of as similar to AJAX or other related models that incorporate a set of application characteristics, including use of specific web technologies and techniques. PWA includes functionality like working offline, push notifications, and device hardware access, enabling creating user experiences similar to native applications on destop and mobile devices. <\/p>\n\n\n\n

PWAs give us the ability to create web apps that take benefit of both web and native app features. Web apps are more discoverable than native apps, it\u2019s a lot easier and fater to visit a website than to install an application, and you can also share web apps by simply sending a link. On the other hand, native apps are better integrated with the operating system and therefore offer a more seamless experience for the users. You can install a native app so that it works offline, and users can easily access their favorite apps through icons, rather than navigating to it using a browser. <\/p>\n\n\n\n

There are many commonly used technologies to create progressive web apps. To call a Web App a PWA, technically speaking it should have the following features: It must run under Secure contexts (HTTPS), must implement a Service Worker, and a manifest file.<\/p>\n\n\n\n

The PWA approach is characterized by the following set of attributes:<\/h3>\n\n\n\n
  1. Responsive<\/li>
  2. Connectivity independent<\/li>
  3. App-like interactions<\/li>
  4. Fresh<\/li>
  5. Safe<\/li>
  6. Discoverable<\/li>
  7. Re-engageable<\/li>
  8. Installable<\/li>
  9. Linkable<\/li><\/ol>\n\n\n\n

    Benefits of PWA<\/h3>\n\n\n\n
    • Cheaper than other apps<\/li>
    • Count towards SEO<\/li>
    • lower usage threshold<\/li>
    • not dependent on app stores<\/li><\/ul>\n\n\n\n

      Disadvantages of PWA<\/h3>\n\n\n\n
      • Fewer functionalities and therefore less UX<\/li>
      • No access to innovations (Such as Bluetooth or NFC)<\/li>
      • No extra room for business model <\/li><\/ul>\n\n\n\n
